什么是 Frontend Slides？

一个 Claude Code Skill，帮助非设计师创建漂亮的网页演示文稿。

[01] 零依赖

单个 HTML 文件，内联 CSS/JS。无需 npm，无需构建工具，开箱即用。

[02] 视觉发现

不知道想要什么风格？生成预览让你选择，而不是抽象的描述。

[03] PPT 转换

将现有的 PowerPoint 文件转换为网页版，保留所有内容和图片。

[04] 独特设计

避免千篇一律的"AI 审美"，每个演示都感觉是量身定制。

cat core_philosophy.md

核心哲学

  • 零依赖 — 单个 HTML 文件胜过千言万语。10年后仍然可用。
  • Show, Don't Tell — 人们不知道自己想要什么，直到他们看到它。
  • Distinctive Design — 每个演示都应该感觉定制，而非模板生成。
  • Production Quality — 代码应该有良好注释、可访问、高性能。
  • Viewport Fitting — 每页幻灯片必须完全适配视口，禁止内部滚动。
// 黄金法则
Each slide = exactly one viewport height (100vh/100dvh)
Content overflows? → Split into multiple slides
Never scroll within a slide.

Clamp() Responsive

viewport 相对单位
完美适配所有屏幕

/* Viewport Fitting - 关键 CSS */
.slide {
  width: 100vw;
  height: 100vh; /* 精确视口高度 */
  height: 100dvh; /* 移动端适配 */
  overflow: hidden; /* 禁止溢出 */
  scroll-snap-align: start;
}

代码示例

<!-- 幻灯片结构 -->
<section class="slide">
  <div class="slide-content">
    <h2 class="reveal">Slide Title</h2>
    <p class="reveal">Content here...</p>
  </div>
</section>

/* 动画 - 滚动触发 */
.reveal {
  opacity: 0;
  transform: translateY(30px);
  transition: all 0.6s cubic-bezier(0.16, 1, 0.3, 1);
}
.slide.visible .reveal {
  opacity: 1;
  transform: translateY(0);
}

cat best_practices.md

最佳实践

内容密度限制（每页）

  • 标题页：1 标题 + 1 副标题 + 可选标语
  • 内容页：1 标题 + 4-6 要点 或 1 标题 + 2 段落
  • 功能网格：1 标题 + 最多 6 卡片 (2x3 或 3x2)
  • 代码页：1 标题 + 最多 8-10 行代码

样式规范

  • 字体大小使用 clamp() 函数
  • 间距使用 viewport 相对单位
  • 图片最大高度 min(50vh, 400px)
  • 内容超出 → 拆分成多页，绝不允许滚动

./troubleshoot.sh

故障排除

[ERR] 字体未加载

检查 Fontshare/Google Fonts URL
确保 CSS 中的字体名称匹配

[ERR] 动画不触发

验证 Intersection Observer 是否运行
检查 .visible 类是否正确添加

[WARN] 滚动吸附失效

确保 html/body 上有 scroll-snap-type
每页 slide 需要 scroll-snap-align: start

[WARN] 移动端问题

在 768px 断点禁用复杂效果
测试触摸事件，减少粒子数量

/* 性能优化 */
@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; }
}
